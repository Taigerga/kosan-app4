Deskripsi Use Case Aplikasi Kosan (Berdasarkan usecase1.txt)

1. **Use Case Name**: Login  
   **Goal in context**: Pengguna dapat masuk ke sistem untuk mengakses fitur sesuai role.  
   **Preconditions**: Pengguna memiliki akun terdaftar.  
   **Successful End Condition**: Pengguna berhasil login dan diarahkan ke dashboard sesuai role.  
   **Failed End Condition**: Login gagal karena kredensial salah atau akun tidak ada.  
   **Primary Actor**: Pengguna/Penghuni/Pemilik  
   **Secondary Actor**: Sistem  
   **Main Flow**:  
     1. Aktor mengakses halaman login '/login'.  
     2. Aktor memasukkan email dan password.  
     3. Sistem memverifikasi kredensial.  
     4. Sistem redirect ke dashboard sesuai role.  
   **Extensions**: Jika kredensial salah, tampilkan pesan error.

2. **Use Case Name**: Register  
   **Goal in context**: Pengguna baru dapat mendaftar akun.  
   **Preconditions**: Tidak ada.  
   **Successful End Condition**: Akun baru dibuat dan diarahkan ke login.  
   **Failed End Condition**: Registrasi gagal karena data tidak valid atau email sudah ada.  
   **Primary Actor**: Pengguna  
   **Secondary Actor**: Sistem  
   **Main Flow**:  
     1. Aktor mengakses halaman register '/register'.  
     2. Aktor mengisi form nama, email, password.  
     3. Sistem menyimpan data user baru.  
     4. Sistem redirect ke login.  
   **Extensions**: Jika email sudah terdaftar, tampilkan error.

3. **Use Case Name**: Logout  
   **Goal in context**: Pengguna keluar dari sistem.  
   **Preconditions**: Pengguna sudah login.  
   **Successful End Condition**: Session dihapus dan diarahkan ke home.  
   **Failed End Condition**: Tidak ada (biasanya selalu sukses).  
   **Primary Actor**: Penghuni/Pemilik  
   **Secondary Actor**: Sistem  
   **Main Flow**:  
     1. Aktor klik logout '/logout'.  
     2. Sistem menghapus session.  
     3. Sistem redirect ke home.  
   **Extensions**: Tidak ada.

4. **Use Case Name**: Melihat Home Page  
   **Goal in context**: Pengguna melihat halaman utama aplikasi.  
   **Preconditions**: Tidak ada.  
   **Successful End Condition**: Halaman utama ditampilkan dengan info kos populer.  
   **Failed End Condition**: Halaman tidak dimuat (server error).  
   **Primary Actor**: Pengguna  
   **Secondary Actor**: Sistem  
   **Main Flow**:  
     1. Aktor mengakses '/'.  
     2. Sistem menampilkan halaman utama dengan informasi kos populer.  
   **Extensions**: Jika kos tidak ada, tampilkan pesan kosong.

5. **Use Case Name**: Mencari Kos  
   **Goal in context**: Pengguna mencari kos berdasarkan kriteria.  
   **Preconditions**: Tidak ada.  
   **Successful End Condition**: Daftar kos sesuai pencarian ditampilkan.  
   **Failed End Condition**: Pencarian tidak menemukan hasil.  
   **Primary Actor**: Pengguna  
   **Secondary Actor**: Sistem  
   **Main Flow**:  
     1. Aktor mengakses '/kos'.  
     2. Sistem menampilkan daftar kos (KosController::index).  
     3. Aktor bisa mencari berdasarkan nama, lokasi, dll.  
   **Extensions**: Jika tidak ada hasil, tampilkan pesan "Tidak ada kos ditemukan".

6. **Use Case Name**: Filter Kos  
   **Goal in context**: Pengguna memfilter kos berdasarkan kriteria.  
   **Preconditions**: Halaman '/kos' sudah diakses.  
   **Successful End Condition**: Kos yang sesuai filter ditampilkan.  
   **Failed End Condition**: Filter tidak menghasilkan hasil.  
   **Primary Actor**: Pengguna  
   **Secondary Actor**: Sistem  
   **Main Flow**:  
     1. Pada halaman '/kos', aktor menggunakan filter (harga, fasilitas, dll.).  
     2. Sistem menampilkan kos yang sesuai filter.  
   **Extensions**: Jika filter kosong, tampilkan semua kos.

      3. Sistem menyimpan pengajuan (PenghuniKontrak::store).  
      4. Sistem kirim notifikasi ke pemilik.  
    **Extensions**: Jika kosId invalid, error.
