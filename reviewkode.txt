5. REKOMENDASI PERBAIKAN UMUM
================================================================================

--- SECURITY IMPROVEMENTS ---

1. INPUT SANITATION:
   - Gunakan strip_tags() untuk text inputs
   - Gunakan htmlspecialchars() untuk output di blade
   - Validasi file upload untuk MIME types (bukan cuma extension)

2. CSRF PROTECTION:
   - Sudah ada @csrf di forms (BAIKUS)
   - Pastikan semua AJAX request memiliki X-CSRF-TOKEN header

3. RATE LIMITING:
   - Tambah rate limiting ke login routes:
     Route::middleware(['throttle:5,1'])->post('/login');
   
4. PASSWORD SECURITY:
   - Gunakan Hash::check() dengan proper pepper jika perlu
   - Implementasi password reset flow
   - Validasi password complexity (min 8 chars, mixed case, number, special char)

5. FILE UPLOAD SECURITY:
   - Validate file MIME type di server side:
     'file' => 'required|file|mimes:jpeg,png,jpg,gif|max:2048|dimensions:min_width=300,min_height=300'
   - Generate unique filename untuk prevent overwrite
   - Store di secure location di luar public root

6. SQL INJECTION PROTECTION:
   - Gunakan parameter binding (Laravel Eloquent sudah aman)
   - Tidak gunakan raw SQL queries tanpa parameter binding

--- PERFORMANCE IMPROVEMENTS ---

1. QUERY OPTIMIZATION:
   - Gunakan eager loading (with()) untuk relationships
   - Gunakan select() hanya field yang dibutuhkan
   - Implementasikan caching untuk expensive queries:
     Cache::remember('kos_popular', 3600, function() {
         return Kos::withCount('reviews')->orderBy('reviews_count', 'desc')->take(5)->get();
     });

2. IMAGE OPTIMIZATION:
   - Gunakan lazy loading untuk gambar di list view
   - Implementasikan image thumbnails (multiple sizes)
   - Tambah WebP format support

3. JAVASCRIPT OPTIMIZATION:
   - Defer non-critical JavaScript
   - Minimize inline JavaScript atau pindah ke file terpisah
   - Gunakan event delegation untuk dynamic elements

4. PAGINATION:
   - Pastikan pagination menggunakan simplePaginate() untuk large datasets
   - Implementasikan infinite scroll sebagai alternatif

--- CODE QUALITY ---

1. TYPE HINTING:
   - Tambah PHPDoc blocks untuk semua methods
   - Gunakan typed properties di PHP 8.1+:
     public function index(): JsonResponse

2. MAGIC STRINGS:
   - Gunakan constants untuk string yang sering diulang:
     const STATUS_ACTIVE = 'aktif';
     const STATUS_PENDING = 'pending';

3. SINGLE RESPONSIBILITY PRINCIPLE:
   - Pindahkan business logic dari controller ke Service class
   - Contoh: KontrakService untuk handle approval/rejection logic

4. TESTABILITY:
   - Buat FormRequest classes untuk validation
   - Tambah unit tests untuk critical paths
   - Gunakan factory untuk test data generation

--- ERROR HANDLING STANDARDIZATION ---

1. GLOBAL ERROR HANDLING:
   - Tambah try-catch wrapper untuk semua controller methods yang berinteraksi dengan external services
   - Implementasikan global exception handler di app/Exceptions/Handler.php
   
2. LOGGING STANDARDIZATION:
   - Gunakan Log::info(), Log::error(), Log::warning() secara konsisten
   - Tambah context ke logs: Log::error('Payment failed', ['id_pembayaran' => $id, 'error' => $e->getMessage()])
   
3. USER-FRIENDLY ERROR MESSAGES:
   - Tampilkan error messages yang user-friendly dalam bahasa Indonesia
   - Tambah action suggestions dalam error messages (e.g., "Coba upload ulang" untuk file upload error)
